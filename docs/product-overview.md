# Canopy Stories Product Overview

Date: 2026-03-27

## Purpose

Canopy Stories is the success-story production product for adult education schools.

It should help Canopy and client schools:

- collect story source material
- generate multi-channel written content
- generate supporting graphics and short-form video
- package deliverables for practical use across web, newsletter, social, and outreach channels

## Core Workflow

1. create a client project or campaign
2. send or host story intake forms
3. receive submissions from school staff or participants
4. generate content drafts
5. generate assets and videos
6. review and package outputs
7. deliver or publish outputs through downstream Canopy products and services

## Main Users

Internal users:

- Canopy staff
- operators managing story production
- content/review staff

External users:

- school admins
- communications staff
- program staff submitting source material

## Product Boundary

Canopy Stories should own:

- projects
- forms
- submissions
- stories
- generated content
- generated assets
- package assembly

It should not own:

- platform sign-in
- workspace membership
- platform entitlements
- global provisioning

## Why It Is A Good Product 2 Candidate

- there is already a meaningful reference implementation
- it fits the current Canopy external-product launch model
- it has a strong standalone workflow boundary
- it is easier to keep separate from platform core than newsletter or WordPress plugin products
